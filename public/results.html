1. Screenshot Capture

Endpoint: POST /v1/screenshot/capture. Tests validate clean mode, pixel-perfect mode, resource policies, and sticky header preservation across diverse web environments.

1.1 The Guardian — Cookie Wall Removal

https://www.theguardian.com/international

Configuration: clean: true, resourcePolicy: "balanced", preserveStickyHeaders: true

The Guardian presents one of the most aggressive cookie consent walls on the web. RenderKit's auto-clean engine successfully removed the full-screen overlay, the top-banner advertisement container, and the floating newsletter signup modal while preserving the article content and layout. Smart Wait ensured the dynamically loaded DOM was fully stable before capture.

1.2 BBC — Sticky Header Preservation

https://www.bbc.com

Configuration: clean: true, preserveStickyHeaders: true, resourcePolicy: "balanced"

BBC presents a difficult cleanup scenario because its sticky navigation header uses a high z-index value that naive heuristic cleaners treat as intrusive. RenderKit's updated preserveStickyHeaders heuristic correctly identifies the BBC header as structural site chrome (top-anchored, full-width, role: navigation) and preserves it, while still removing the cookie notice and any floating overlays.

1.3 CNN — Heavy JS Stabilization

https://edition.cnn.com

Configuration: clean: true, resourcePolicy: "balanced"

CNN is one of the heaviest JavaScript environments on the web, loading hundreds of dynamic ad slots, tracking scripts, and multimedia content. RenderKit's Smart Wait Engine successfully analyzed DOM stability over time and captured only after the page fully settled. The result is a clean, readable snapshot without half-loaded ad containers or layout shifts.

4. Integration Examples

RenderKit exposes a single RESTful interface across all language environments. Below are accurate, tested code snippets for three major runtimes used in production.

LanguageEnvironmentCode Snippet
cURLCLI / Shellcurl -X POST https://api.renderkit.com/v1/screenshot/capture -H "x-api-key: YOUR_KEY" -d '{"url":"https://example.com","clean":true}'
Node.jsNode 22+fetch("https://api.renderkit.com/v1/screenshot/capture",{method:"POST",headers:{"x-api-key":"YOUR_KEY","Content-Type":"application/json"},body:JSON.stringify({url:"https://example.com",clean:true})}).then(r=>r.json()).then(console.log)
Python 3Python 3.9+import requests; r = requests.post("https://api.renderkit.com/v1/screenshot/capture", json={"url":"https://example.com","clean":true}, headers={"x-api-key":"YOUR_KEY"}); print(r.json())
