RenderKit API

Production-grade browser rendering infrastructure for screenshots, PDFs, and Open Graph images. RenderKit is designed for reliable server-side captures with smart page stabilization, clean-mode heuristics, and API gateway controls.

Quickstart

Render your first screenshot.

Send a JSON payload to the capture endpoint with your API key. By default, RenderKit returns JSON with base64-encoded content and render metadata. Send an image or PDF Accept header when you need binary output directly.

curl -X POST https://api.renderkit.com/v1/screenshot/capture
-H "Content-Type: application/json"
-H "x-api-key: YOUR_API_KEY"
-d '{ "url": "https://example.com", "width": 1280, "height": 800, "format": "png", "wait": "smart", "clean": true }'
API Reference

Endpoints

POST/v1/screenshot/capture

Screenshot Capture

Capture a URL, raw HTML document, or server-side template as a PNG or JPEG. Supports smart stabilization, full-page capture, custom viewport dimensions, custom CSS, clean mode, animation freezing, color scheme emulation, custom headers, and debug telemetry.

{ "url": "https://example.com", "width": 1280, "height": 800, "fullPage": false, "format": "jpeg", "quality": 70, "wait": "smart", "clean": true, "debug": false }
POST/v1/screenshot/pdf

PDF Generation

Convert URLs, raw HTML, or templates into print-ready PDF documents. Ideal for reports, invoices, statements, confirmations, receipts, and internal automation workflows.

{ "url": "https://example.com/report", "format": "A4", "landscape": false, "printBackground": true, "clean": false, "debug": false }
POST/v1/screenshot/og

Open Graph Image Generation

Generate a 1200 by 630 PNG social preview image from structured text, theme values, and optional logo input. Useful for dynamic article cards, product launches, changelogs, dashboards, and campaign previews.

{ "title": "Launch faster with RenderKit", "description": "Screenshots, PDFs, and OG images from one API.", "theme": "dark", "background_color": "#000000", "logo_url": "https://example.com/logo.png" }
GET/health

Health Check

Returns service status and version metadata. This endpoint does not require authentication.

Request Model

Common parameters

Input sources

Use url for live pages, html for raw HTML documents, or template with data for server-side template rendering.

Rendering controls

Set width, height, fullPage, wait, waitFor, waitForSelector, format, quality, and colorScheme to shape output behavior.

Page processing

Enable clean, freezeAnimations, stealth, custom css, or request headers to control page preparation before capture.

Diagnostics and cache

Use debug for render telemetry and noCache when you need a fresh browser render instead of a cached response.

Responses

JSON and binary output

Default JSON response

Render endpoints return success, base64 content, render_time, request_id, and timestamp. Screenshots and OG images use image_base64. PDFs use pdf_base64.

Binary response

Use Accept: image/png, Accept: image/jpeg, or Accept: application/pdf to receive the rendered file directly.

Platform Notes

Operational behavior

Gateway-first access

Production traffic should use the public API gateway with x-api-key. Direct backend access is protected by an internal gateway secret.

Caching

Equivalent render requests are cached to reduce latency and browser work. Use noCache: true when fresh output is required.

Smart stabilization

The smart wait engine waits for fonts, lazy content, and DOM stability before capture, improving output consistency for SPAs and dynamic pages.

Error model

Validation errors return 400, authentication failures return 401, and render failures return structured JSON with render_failed.
